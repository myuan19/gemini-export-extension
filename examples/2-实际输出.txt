### 复杂解析测试：Windows 环境部署


本示例旨在测试解析器对 Windows 风格路径（`C:\Users\Name`）与 Markdown 语法混合时的识别能力。


### 1. 系统环境初始化


- **配置文件路径定位**
系统会优先读取用户目录下的隐藏配置。请确认以下路径存在：
`C:\Users\Administrator\.config\settings.yaml`
如果该路径不存在，程序将回退至安装目录：

    ```PowerShell
    $DefaultPath = "C:\Program Files\App\DefaultConfig\"
    # 检查并创建路径
    New-Item -Path $DefaultPath -ItemType Directory -Force
    ```

    *上述代码块演示了如何在 PowerShell 中处理带有空格的 Windows 路径。*
- **动态路径拼接逻辑**
    1. **临时目录清理**
在执行任务前，需清理以下位置的缓存：
    - **用户缓存**
路径通常为：`%LOCALAPPDATA%\Temp\Inbound\Log\`
执行清理的代码片段：

            ```Python
            import os
            # 测试原始字符串处理路径
            target_dir = r"C:\Windows\Temp\Direct_Drop\v1.2"
            print(f"Cleaning: {target_dir}")
            ```

            *注意：在 Python 中使用 `r""` 前缀可以避免反斜杠被误认为转义字符。*
    - **系统日志归档**
归档文件将被移动到：`D:\Backup\Archive\2026_Logs\`


### 2. 注册表与批处理脚本


在 Windows 自动化中，路径经常与命令参数混合。请测试下方的解析效果：


- **注册表项导出**
使用以下命令备份特定的注册表分支：

    ```DOS
    reg export "HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion" C:\RegBackup\system_core.reg /y
    ```

    导出的文件应保存在 `C:\RegBackup\` 目录下。
- **混合缩进测试**
    - **子任务 A**
当路径包含特殊字符（如空格或括号）时：

        ```python
        @echo off
        set "WORK_DIR=C:\Program Files (x86)\Common Files\System"
        cd /d "%WORK_DIR%"
        dir > "C:\Users\Public\Documents\file_list.txt"
        ```

        *这段文字紧跟在批处理脚本后，用于测试 Windows 路径在双引号内的解析稳定性。*


### 3. 部署状态汇总


| **模块** | **预设路径** | **权限要求** |
| --- | --- | --- |
| 内核 | `C:\Windows\System32\drivers\` | **System** |
| 插件 | `D:\AppData\Roaming\Plugins\` | *User* |


---


**解析验证点：**


1. **反斜杠识别**：确保路径中的 `\` 不会被误认为是 Markdown 的转义符号（例如 `\.` 不应只显示 `.`）。
2. **长路径换行**：在列表缩进中，观察长路径如 `C:\Users\Administrator\.config\settings.yaml` 是否会导致布局崩溃。
3. **代码块上下文**：检查代码块前后的 Windows 路径描述文字是否能保持正确的缩进层级。
